# Supabase Setup Guide

This document covers everything you need to configure Supabase sync for TimeTracker,
including the iOS Safari Content Blocker integration.

---

## 1. Create a Supabase Project

1. Go to https://supabase.com and create a new project.
2. Note your **Project URL** and **anon public API key** (Settings → API).

---

## 2. Create the Required Tables

Run the following SQL in the **SQL Editor** (https://app.supabase.com → SQL Editor).

### time_entries

```sql
CREATE TABLE IF NOT EXISTS time_entries (
  id                BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  local_id          BIGINT,          -- original SQLite row id from the source device
  device_id         TEXT NOT NULL,   -- unique per device, prevents cross-device duplicates
  user_id           TEXT,
  task_name         TEXT NOT NULL,
  project           TEXT,
  start_time        TEXT NOT NULL,
  end_time          TEXT,
  duration_seconds  BIGINT DEFAULT 0,
  tags              TEXT,
  notes             TEXT,
  billable          BOOLEAN DEFAULT false,
  hourly_rate       NUMERIC DEFAULT 0,
  created_at        TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE (device_id, start_time, task_name)
);

-- Index for fast pulls filtered by device
CREATE INDEX IF NOT EXISTS idx_time_entries_device_id ON time_entries(device_id);
CREATE INDEX IF NOT EXISTS idx_time_entries_start_time ON time_entries(start_time);
```

### blocked_sites (ground truth for all devices)

```sql
CREATE TABLE IF NOT EXISTS blocked_sites (
  id         BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  domain     TEXT NOT NULL UNIQUE,
  enabled    BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Index for fast domain lookups
CREATE INDEX IF NOT EXISTS idx_blocked_sites_domain ON blocked_sites(domain);
```

## 3. Row Level Security (RLS)

Enable RLS so only authenticated users (or service-role API calls using your
anon key) can read/write.

```sql
-- time_entries
ALTER TABLE time_entries ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow anon read/write" ON time_entries
  FOR ALL
  TO anon
  USING (true)
  WITH CHECK (true);

-- blocked_sites
ALTER TABLE blocked_sites ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow anon read/write" ON blocked_sites
  FOR ALL
  TO anon
  USING (true)
  WITH CHECK (true);
```

> **Note:** The above policies allow full public access via the anon key.
> For a multi-user or shared deployment, tighten these with auth-based policies.

---

## 4. Configure the App

Open **Settings → Sync** inside TimeTracker and fill in:

| Field | Value |
|-------|-------|
| Supabase URL | `https://<your-project-ref>.supabase.co` |
| API Key | your **anon** (public) key |
| Time entries table | `time_entries` (default) |
| Blocked sites table | `blocked_sites` (default) |
| Auto-sync | ✓ enabled (syncs every 5 minutes) |

Click **Test Connection** to verify.

## 5. iOS Safari Content Blocker Setup (Xcode)

The Safari Content Blocker is a native iOS App Extension that actually enforces
blocked sites in Safari.  You need Xcode to finish this setup.

### 5a. Register the App Group in Apple Developer Console

1. Log in at https://developer.apple.com/account.
2. Go to **Certificates, Identifiers & Profiles → Identifiers**.
3. Select your main app ID `com.timetracker.app` → **Edit** → enable
   **App Groups** → add `group.com.timetracker.shared`.
4. Create a new App ID `com.timetracker.app.blocker` with:
   - Description: `TimeTracker Blocker`
   - Capabilities: **App Groups** → `group.com.timetracker.shared`
5. Regenerate your provisioning profiles for both identifiers.

### 5b. Regenerate the Xcode Project

The `project.yml` (XcodeGen config) already has the `TimeTrackerBlocker` target.
Run from `src-tauri/gen/apple/`:

```bash
xcodegen generate
```

This regenerates `timetracker-app.xcodeproj` with the extension target included.

### 5c. Set Your Development Team

In Xcode, select each target (`timetracker-app_iOS` and `TimeTrackerBlocker`)
and set **Signing & Capabilities → Team** to your Apple developer team.

Or set it once in `src-tauri/tauri.conf.json`:
```json
"iOS": {
  "developmentTeam": "YOUR_TEAM_ID_HERE"
}
```

### 5d. Assign Provisioning Profiles

In **Build Settings** for both targets, set:
- `CODE_SIGN_IDENTITY`: `Apple Development` (debug) / `Apple Distribution` (release)
- `PROVISIONING_PROFILE_SPECIFIER`: the profile you created in step 5a

### 5e. Enable the Content Blocker on the Device

After installing the app on your iPhone/iPad:

1. Open **Settings → Safari → Extensions → Content Blockers**
2. Enable **TimeTracker Blocker**

Blocked sites you add in the TimeTracker app will then be enforced in Safari.

## 6. How Sync Works

```
 iPhone / iPad                         Supabase
 ─────────────────────────────         ────────────────────
 SQLite  ──push new entries──►  time_entries table
         ◄─pull remote entries─

 SQLite  ──push local sites───►  blocked_sites table
         ◄─pull (ground truth)─   (Supabase wins on conflict)
         │
         ▼
  blocked_rules.json  (app data dir)
         │
         ▼  BlockerManager.swift copies on foreground
  App Group container: group.com.timetracker.shared
         │
         ▼  SFContentBlockerManager.reload
  Safari Content Blocker active ✓
```

### Sync triggers
- **Manual**: Settings → Sync → "Sync Now"
- **Auto**: every 5 minutes when auto-sync is enabled
- **On foreground**: `BlockerManager.syncAndReload()` is called automatically
  via the `UIApplicationDidBecomeActiveNotification` observer

## 7. Troubleshooting

| Symptom | Fix |
|---------|-----|
| "Supabase not configured" | Enter URL and key in Settings → Sync |
| "blocked_sites_table not configured" | Set the field or leave blank for default `blocked_sites` |
| Safari still loads blocked sites | Check Settings → Safari → Extensions → Content Blockers is ON |
| Extension not visible in Settings | Rebuild app with Xcode after running `xcodegen generate` |
| Rules don't update after adding a site | Pull-to-refresh or wait for next auto-sync, then background → foreground the app |
| App Group not available | Confirm group ID is registered in Apple Developer Console for both app IDs |
