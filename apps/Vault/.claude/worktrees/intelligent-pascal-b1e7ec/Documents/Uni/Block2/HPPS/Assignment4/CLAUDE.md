## Technical Details

### Data Structures Required

**For id_query_indexed.c:**
```c
struct index_record {
    int64_t osm_id;
    const struct record *record;
};
```
- Linear search but on compact index (better spatial locality)

**For id_query_binsort.c:**
- Same index_record structure
- Sort by osm_id using qsort()
- Binary search for O(log n) lookup

**For coord_query_naive.c:**
- Euclidean distance: sqrt((x1-x2)^2 + (y1-y2)^2)
- Linear search for minimum distance

### Locality Analysis Framework
- **Temporal locality:** Same data accessed multiple times recently
- **Spatial locality:** Accessing nearby memory addresses
- **Cache line:** Typically 64 bytes on modern systems

### Key Insights for Report
- `struct record` is LARGE (contains strings, many fields) → poor spatial locality when scanning
- `struct index_record` is SMALL (16 bytes: int64_t + pointer) → better spatial locality
- Binary search has poor temporal locality (jumps around) but O(log n) beats O(n)
- Sorted index: build time O(n log n), query time O(log n)
- Naive: build time O(1), query time O(n)
